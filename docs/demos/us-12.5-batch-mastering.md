# US-12.5: Batch mastering

*2026-06-21T01:34:16Z*

US-12.5 lets a musician master many clips with one profile/service in a single request, each tracked as an individual mastering job under one batch id. Rather than a new `MasteringBatch` document (the stale issue plan), it reuses the as-built US-10.5 `BatchJob` model with `operation="mastering"` and the as-built per-service mastering credit costs (dolby 3 / landr 2 / bakuage 5). This demo drives the real FastAPI app over a local MongoDB (httpx ASGITransport), seeding via Beanie exactly like the integration tests.

Setup: seed one user with 100 credits and 5 source clips.

```bash
uv run python .cache/us125_demo.py seed 2>/dev/null
```

```bash
uv run python .cache/us125_demo.py submit 2>/dev/null
```

**AC4** (individual failures do not halt the batch) — a request mixing one owned clip and one unknown clip still returns 202: the owned clip is queued, the unknown clip is recorded as a failed entry (job_id null + reason), and only the owned clip is charged.

```bash
uv run python .cache/us125_demo.py partial 2>/dev/null
```


**AC5** (exceeding the batch limit returns 422) — a request with 21 clip ids (limit is 20) is rejected by request validation before any credit moves.

```bash
uv run python .cache/us125_demo.py limit 2>/dev/null
```

```output
POST /api/v1/mastering/batch  clips=21 (limit is 20)  ->  HTTP 422
  error: too_long on ['body', 'clip_ids'] -> List should have at most 20 items after validation, not 21
```

Credit gate (functional requirement: credits deducted per clip) — a batch the user cannot afford (needs 9.0, has 5.0) is rejected with 402 and an {error, balance, required} payload; no credits move and no jobs are created (all-or-nothing upfront).

```bash
uv run python .cache/us125_demo.py credits 2>/dev/null
```

```output
POST /api/v1/mastering/batch  clips=3 service=dolby (needs 9.0, has 5.0)  ->  HTTP 402
{
  "detail": {
    "error": "insufficient_credits",
    "balance": 5.0,
    "required": 9.0
  }
}

-> credits unchanged: 5.0 -> 5.0; jobs created: 0 (all-or-nothing gate)
```
