# US-13.1: AI Cover Art Generation

*2026-06-21T03:38:52Z*

US-13.1 adds cover-art generation, selection and custom upload to the platform API under `/api/v1/clips/{id}/artwork`. This demo drives the **real** HTTP API end-to-end over a local MongoDB and on-disk storage with a **real** background JobProcessor. Only the OpenAI DALL-E client is faked (no API key is configured here — mock-verified, like the Dolby mastering integration), so artwork generation exercises the genuine async job → upscale → store → ArtworkOption pipeline.

The driver creates a clip titled *Midnight Drive* with style tags `synthwave, retro, neon`, then walks each acceptance criterion: generate 4 options, select one, upload custom art, and reject undersized/corrupt uploads. Every step prints its HTTP status and the resulting outcome (not just "exited 0").

```bash
uv run python scripts/demo_us_13_1_artwork.py 2>/dev/null
```
