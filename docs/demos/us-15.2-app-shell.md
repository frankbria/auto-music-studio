# US-15.2: Application Shell Layout

*2026-06-25*

The four-zone application shell (sidebar · main · contextual right panel · fixed bottom playbar) is wired into the Next.js App Router root layout so it persists across every route. Panels are placeholders — real navigation (US-15.3) and player controls (US-15.6) land later.

Each scenario below drives the **real** Next.js app: the dev server is started (`npm run dev`), a headless Chromium loads `http://localhost:3000` at the target viewport, and the shell's DOM is probed with `getBoundingClientRect()` / `getComputedStyle()`. The JSON shown is the actual measured layout — nothing is mocked. Screenshots are in the PR.

## AC1 — Layout renders sidebar + main + bottom playbar on all routes

Probed at 1024, 1440, and 1920px. The shell is present, there is exactly one `<main>` landmark, and the sidebar is a fixed 64px column with main starting immediately after it.

```output
=== 1024x768 ===
{"hasShell":true,"sidebar":{"x":0,"w":64},"main":{"x":64,"w":960},"mainCount":1}
=== 1440x900 ===
{"hasShell":true,"sidebar":{"x":0,"w":64},"main":{"x":64,"w":1376},"mainCount":1}
=== 1920x1080 ===
{"hasShell":true,"sidebar":{"x":0,"w":64},"main":{"x":64,"w":1856},"mainCount":1}
```

## AC2 — Right panel visible on routes that provide contextual content

The right panel renders only when a route passes `rightPanel` content to `AppShell`. With content supplied, at ≥ lg it is a 320px `flex` column; below lg it is `display:none`.

```output
=== rightpanel @1440 (content provided) ===
{"rightPanelInDom":true,"rightPanelDisplay":"flex","rightPanelVisible":true,"rightPanelWidth":320}
=== rightpanel @900 (below lg breakpoint) ===
{"rightPanelInDom":true,"rightPanelDisplay":"none","rightPanelVisible":false,"rightPanelWidth":0}
```

## AC3 — Sidebar does not overlap main content

The sidebar's right edge equals the main region's left edge at every width — flex layout, no overlap.

```output
sidebar.right == main.left (64px) at 1024 / 1440 / 1920  -> sidebarOverlapsMain: false
```

## AC4 — Bottom playbar remains visible during scroll

The playbar is rendered as a sibling of `AppShell` in the root layout (not inside the shell's `overflow-hidden` container) so its `position: fixed` stays viewport-anchored regardless of any future ancestor transform. `AppShell` reserves `padding-bottom: 72px` so content is never hidden behind it.

```output
=== playbar @1440 ===
{"playbarInDom":true,"position":"fixed","zIndex":"50","bottom":900,"width":1440,"isSiblingOfShell":true}
```

## AC5 — Layout usable at 1024px, 1440px, and 1920px

All of the above probes pass at all three widths: shell present, single main landmark, no sidebar/main overlap, playbar fixed and full-width, right panel correctly shown/hidden by breakpoint.
