# US-4.4: Import Audio Files into Workspace

*2026-04-13T04:01:35Z*

US-4.4 adds the `acemusic import` command so musicians can bring existing audio files into their workspace for use as references in covers, remixes, and mashups. Supported formats: WAV, FLAC, MP3, OGG, AAC, AIFF. Imported clips are registered in the SQLite metadata database with BPM and key auto-detected via librosa.

Acceptance Criterion 1: The import command is accessible and documents its arguments.

```bash
uv run acemusic import --help
```

```output
                                                                                
 Usage: acemusic import [OPTIONS] FILE_PATH                                     
                                                                                
 Import an existing audio file into the active workspace.                       
                                                                                
╭─ Arguments ──────────────────────────────────────────────────────────────────╮
│ *    file_path      PATH  Path to the audio file to import. [required]       │
╰──────────────────────────────────────────────────────────────────────────────╯
╭─ Options ────────────────────────────────────────────────────────────────────╮
│ --title        TEXT  Title for the imported clip (defaults to filename).     │
│ --help               Show this message and exit.                             │
╰──────────────────────────────────────────────────────────────────────────────╯

```

Acceptance Criterion 2: Importing a WAV file copies it to the workspace and creates a metadata record. We use the bundled epic_theme.mp3 as our test file.
